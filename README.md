# vnet - заметки по разработке

## Проблема: не приходит ICMP ответ
- После загрузки модуля (sudo insmod vnet.ko) и создания маршрута (sudo ip route add 10.0.0.1/32 dev vnet0)
- При попытке пропинговать интерфейс ожидается ответ от интерфейса
- Гипотеза: выполнение не доходит до формирования ответа, удаляет пришедший запрос и выходит
- Следующий шаг: выяснить куда доходит выполнение

## Ход исследования ошибки
- Добавлены отладочный лог модуля
- По результатам отладки выполнение не проходит условие (icmph->type == ICMP_ECHO && iph->daddr == vnet_ip_be32) для формирования ICMP ответа
- Следующий шаг: проверить условия на ложность

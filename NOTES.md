## Проблема: неправильный порядок байт в IP-адресах.
- После загрузки модуля (sudo insmod vnet.ko) и создания маршрута (sudo ip route add 10.0.0.1/32 dev vnet0)
- При попытке пропинговать интерфейс ожидается правильный порядок байт IP-адреса в логах модуля ядра (sudo dmesg | grep "vnet")
- Попробовал `ip_hdr(skb)` — тот же результат
- Попробовал ручное смещение через `skb->data + sizeof(ethhdr)` — тот же результат
- Гипотеза: возможно Ethernet-заголовка нет и IP начинается с другого смещения
- Следующий шаг: hex-дамп первых 34 байт skb->data
